Driver Abstract and Integration

強化團隊以相同 interface 呼叫接口與工具，抽象化基礎功能套件，如 http-client, snowflake, NSQ 等基礎套件，即便在底層被抽換的情況下，引用套件的服務依然可不受影響，不需要做出額外的調整。具體的案例在 http-client 與 snowflake 的經驗被凸顯出來：

http-client: 由於一開始採用 benchmark 優秀的 fasthttp，當作 http-client 的基底實作，但 fasthttp 存在著 request body data race 的嚴重瑕疵，我們緊急抽換了底層的 fasthttp，過程中相依 http-client 的服務僅需 vendor 新版本，無須調整接口與參數的使用方式，完全符合一開始設計的目標。

snowflake: 此套件應用於分散式系統產生不重複 ID，因應服務的擴容，原有 bwmarrin/snowflake 的 10 Bit NodeID 已無法滿足我們的需求，因此底層抽換為有 16 bits NodeID 的 sony/sonyflake，同樣地相依此套件的服務，無須更換接口與參數，符合我們抽像化套件目的。

Project Lead

於小型專案中執行任務拆分，結合團隊微服務架構，適當的將服務切分至合適的 micro-services，並協調專案參與人統合接口 spec， 確立開發人員可並行開發，以左移精神降低整體開發時程。規劃與抽象遊戲流程，提供適合多人共同開發遊戲框架，開發人員可專注於遊戲邏輯實現，無須費心於連線,存取,角色驗證等基礎實作。

System Trace and Monitoring

運用 ELK 與 Kiali 等工具，追蹤系統錯誤及流量異常，可持續的針對系統特定功能或排程監控，觀察每個任務的執行時間，特別是離群峰值的觀察，做出及時對應的 alert，持續後續的效能瓶頸追蹤。利用 Kiali 於流量視覺化提供我們快速排查的能力，如舊服務需要下線時可透過其提供的圖表，觀察是否還有正在對服務使用的連線與流量。
